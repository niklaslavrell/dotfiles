#
# ## Help ##
#
# Subject line imperative uppercase verbs:
#
#   Add = Create a capability e.g. feature, test, dependency.
#   Drop = Delete a capability e.g. feature, test, dependency.
#   Fix = Fix an issue e.g. bug, typo, accident, misstatement.
#   Bump = Increase the version of something e.g. a dependency.
#   Make = Change the build process, or tools, or infrastructure.
#   Start = Begin doing something; e.g. enable a toggle, feature flag, etc.
#   Stop = End doing something; e.g. disable a toggle, feature flag, etc.
#   Optimize = A change that MUST be just about performance, e.g. speed up code.
#   Document = A change that MUST be only in the documentation, e.g. help files.
#   Refactor = A change that MUST be just refactoring.
#   Reformat = A change that MUST be just format, e.g. indent line, trim space, etc.
#   Rephrase = A change that MUST be just textual, e.g. edit a comment, doc, etc.
#
# Conventional commits cheat sheet:
#
#   feat = A new feature
#   fix = A bug fix
#   style = Formatting, white-space, missing semi-colons, etc.
#   refactor = A code change that neither fixes a bug or adds a feature
#   release = Create a release commit
#   perf = A code change that improves performance
#   ci = CI related changes
#   docs = Documentation only changes
#   test = Adding missing tests or correcting existing tests
#   chore = Dependencies, build process or utility tool changes
#
# ## Inspiration ##
#
# https://github.com/joelparkerhenderson/git-commit-message
# https://github.com/streamich/git-cz
# https://www.conventionalcommits.org/
